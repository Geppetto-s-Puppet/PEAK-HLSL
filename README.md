# Shader Library for PEAK
PEAKのレンダリングパイプラインはURP Forwardのため、通常であればビルド後にカスタムシェーダーを挿入することなど不可能。
* **このプロジェクトためにに勉強したスタック:** Unity, BepInEx, C#, .NET, C++, Win32 API, HLSL, D3D12

### C#(Unity)側
- BepInExを注入し、`.exe`を書き換えることなく、起動時に`.dll`を差し込んで`MonoBehaviour`を継承したコードを動かす。
- リフレクションで直接privateフィールドにアクセスし、パスを追加してから、`UnsafePass`で強引に設定フラグを有効化。
- あとはグローバルテクスチャのポインタを安全に取り出したいので、`WaitForEndOfFrame` でGPUの描画完了を待つだけ。
### C++(DX12)側
- ダミーテクスチャをあえて受信し、`ID3D12Resource::GetDevice()`を経由してUnityが作ったデバイスを横取り、同期。
- Win32 APIでゲームの最前面に重なるウィンドウを作成し、ヒットテストを常に透過扱いにして、非アクティブを強制。
- RenderGraphは内部でリソースをエイリアスしかねないので、毎フレーム自前のリソースへコピーする(形式も変換)。
- 大きな1枚の三角形で画面全体をカバーして最適化(APIコールも減るし、画面外はクリッピングで破棄される)。
- あとはUVのY座標を反転させ、フォーマット変換、リソースバリア管理、GPU同期をちゃちゃっと済ませる。
### HLSL側
- カラー・深度・法線の3バッファがSRVとして登録される。
- `/`キーが押されるとモードを切替。一周すると非表示。

- 本ツールはゲームの改変を意図するものではなく、グラフィクスプログラミングの学習・研究目的で制作されました

# 導入方法
1. BepInEx を PEAK のゲームフォルダに導入する(thunderstoreから起動していれば、自動で生成される)
2. `BepInEx/plugins/` に `com.alex.peakhlsl.plugin.dll` と `com.alex.peakhlsl.native.dll` を配置する
3. ゲームを起動して、`/`キーを押下(タイトル画面でも動くが、マップ画面の方が分かりやすい。
